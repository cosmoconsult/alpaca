# COSMO Alpaca Newsletter 05/2026

Welcome to the second edition of the COSMO Alpaca Newsletter! Here's what's new and noteworthy.

---

## General

- **New Status & Health Dashboard:** A new status & health dashboard is now available, giving you quick visibility into our current system status including the platforms Azure DevOps and GitHub as well as known issues and announcements. Before opening a support ticket, please check the dashboard first and make sure your Pipeline Templates (on Azure DevOps) or AL-Go System Files (on GitHub) are up to date - this helps reduce duplicates and speeds up incident resolution. The dashboard is accessible directly in **Alpaca Home** in the VS Code extension and via **COSMO Docs → COSMO Alpaca → Support → Status & Health**. [→ Check out the dashboard](https://cosmo-ppi.westeurope.cloudapp.azure.com/)

- **COSMO Alpaca Extension out of Preview:** The new COSMO Alpaca VS Code extension has left preview and is now the official **COSMO Alpaca** extension. The previous extension has been renamed to **COSMO Alpaca (legacy)**. Extension logos and icons have been updated accordingly. If you haven't switched yet, now is the time!

- **Clone Container Action:** You can now clone an existing container in the VS Code extension - right-click a stopped container and select **clone** from the context menu to create a new one based on it. The cloned container inherits the settings from the original and starts with a copy of its database.
  
- **Improvements to the Documentation:** General improvements to the Alpaca documentation with a strong focus on the GitHub AL-Go area. Added new GitHub scenarios covering, among others, environment configuration, release deployment, and workflow setup. Also introduced a FAQ page including best practices and answers to frequently asked questions. [→ Read the docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/support/github-al-go-faq.html)

---

## Quality of Life Improvements

- **Auto-Reveal current Repository:** The VS Code extension now automatically identifies the repository of your current workspace and expands the corresponding elements in the tree view.

- **Improved performance for Containers:** The download of NuGet artifacts inside a container was improved by caching the information of already downloaded apps.

---

## Azure DevOps

- **Azure DevOps artifacts feeds in VS Code:** The VS Code extension now lets you browse Azure DevOps artifact feeds directly from the Azure DevOps tree view. This includes viewing feeds, views, package versions, and downloading packages as ZIP.

- **New configuration settings:** We added support for the settings `codeCopsTest` and `rulesetFileTest` in the `cosmo.json`. This enhancement allows you to set these settings specifically for the test project. [→ Read the docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/azure-devops/setup-cosmo-json.html#compiler)

---

## GitHub / AL-Go

- **GitHub Ruleset for required Status Checks:** The repository creation and initialization with COSMO Alpaca now also creates a GitHub ruleset with required status checks based on parameter `RequiredStatusChecks` of the GitHub variable `ALPACA_REPO_STANDARDS` (default: "Pull Request Status Check"). [→ Read the docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/admin/index.html#github-repo-standards)

- **New templates for AppSource Apps:** A new custom AL-Go template **Alpaca-AppSource** (based on AppSource AL-Go template from Microsoft) and a corresponding public GitHub template **Alpaca-AppSource-Template** have been created and integrated in the COSMO Alpaca Extension.

- **Support for custom GitHub templates:** The COSMO Alpaca Extension now supports the creation of repositories from custom GitHub templates of the target organization/owner. [→ Read the docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/github/create-app.html#create-repository)

- **Update to AL-Go v8.3:** The custom AL-Go templates for COSMO Alpaca (**Alpaca-AppSource** and **Alpaca-PTE**) were updated from AL-Go v8.1 to v8.3.
  - Some highlights:
    - Pull Request trigger changed to "pull_request" (before: "pull_request_target")
    - Support for GitHub Merge Queues
    - Overwrite default values of workflow inputs
    - See the release notes for [AL-Go v8.2](https://github.com/microsoft/AL-Go/releases/tag/v8.2) and [AL-Go v8.3](https://github.com/microsoft/AL-Go/releases/tag/v8.3)

- **Do not create unnecessary Containers:** We have optimized the provisioning of image containers so that they are now only created when they are needed. For example, no container is requested if the AL-Go setting `doNotPublishApps` is set to true.

- **Evaluate merging multiple Azure DevOps Repositories into one GitHub Repository:** Since COSMO Alpaca only supports a single app on Azure DevOps, and since AL-Go supports multiple projects with multiple apps, it may be necessary to merge multiple Git repositories into a single one. Because this process is not entirely straightforward, we are providing a blueprint. [→ Read the docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/github/migrate-from-azure-devops.html#merge-multiple-azure-devops-repositories-into-one-multi-project-al-go-repository)
