# COSMO Alpaca Newsletter 03/2026

Welcome to the first edition of the COSMO Alpaca Newsletter! Here's what's new and noteworthy.

---

## General

- **COSMO Alpaca Preview Extension** — The Preview extension will soon replace the legacy extension and come out of preview. We'd love your feedback — try it out and let us know what you think!
  - The new Preview extension now provides comprehensive support for both **GitHub (AL-Go)** and **Azure DevOps** in a single tool — all features from the legacy extension and the PowerApp have been migrated. Choose between a *technical* or *functional* view to match your role, whether you're a developer, consultant, or project manager. [→ See full feature overview](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/index.html)
  - **MCP Tools (Beta):** Built-in Model Context Protocol tools expose almost all COSMO Alpaca functionality to AI agents like GitHub Copilot — create projects, containers, and manage CI/CD through natural language, with zero extra setup. [→ Learn more about MCP Tools](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/shared/mcp-tools.html)
  - **Browser Edition (vscode.dev):** Use COSMO Alpaca directly in the browser via vscode.dev — ideal for consultants and project managers who need quick access without any local installation. Spin up demo environments, do user acceptance testing or create and customize projects — all without installing anything locally. [→ Get started with the Browser Edition](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/getting-started/vscode-dev.html)

- **VS Code NuGet Packages View:** Browse Business Central NuGet feeds, view package details, add custom feeds, and download packages as ZIP or directly into your AL package cache. Available feeds are also usable as dependency artifacts in your containers. [→ Docs for GitHub](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/github/packages-view.html) | [→ Docs for Azure DevOps](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/azure-devops/packages-view.html)

- **Updated Alpaca Documentation:** Our docs are now clearly separated between Azure DevOps and GitHub sections for easier navigation. More GitHub documentation is on the way. [→ Docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/)

---

## Azure DevOps

- **Git Submodule Support in Pipelines:** Pipelines now support checking out Git submodules from repositories within the same Azure DevOps organization — useful when you share files across multiple app repositories. [→ Read the docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/azure-devops/pipeline-variables.html#git-submodules)

- **Page Scripting Tests:** Automatically execute Page Scripting Tests during CI builds by pointing to a test folder via the `PageScriptingTestsDirectory` pipeline variable. Results are shown alongside classic test results in the pipeline's "Tests" tab. [→ Read the docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/azure-devops/pagescriptingtests.html)

---

## GitHub / AL-Go

- **alpaca.json Deprecated:** The `.alpaca/alpaca.json` configuration file is deprecated — settings should now be placed in the standard AL-Go settings files (e.g. `.github/AL-Go-Settings.json`). A migration mapping is available in the docs. [→ Migration guide](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/github/setup-al-go-settings.html)

- **xliff-sync Translation Support:** Automatically create and test translation files (.xlf) in your workflows using xliff-sync. Configure `alpaca.createTranslations`, `alpaca.testTranslations`, and language tags in your AL-Go settings. [→ Read the docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/github/setup-al-go-settings.html#translations)

- **New AL-Go Settings for Containers:** Containers now support the `trustMicrosoftNuGetFeeds` and `trustedNuGetFeeds` setting, aligning container behavior with AL-Go build workflows. Configure trusted NuGet feeds and other AL-Go settings seamlessly across dev and build containers. [→ Read the docs](https://docs.cosmoconsult.com/en-us/cloud-service/alpaca/github/setup-al-go-settings.html#al-go-settings)
